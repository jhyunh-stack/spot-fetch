# SpotFetch — Plan 기능 PRD

**목표**: Spot → Plan 전환율 높이기
**North Star**: Spot 저장 후 Plan 생성까지 가는 비율

---

## 탭 구조

| 탭 | 역할 |
|---|---|
| **Spots** | 릴스에서 수집한 장소 인박스. 읽기 전용. |
| **Plan** | 수집된 Spot으로 여행 계획 수립·편집. |
| **Explore** | 다른 유저 루트 탐색. |

---

## User Flow 전체

```
유저가 Spots 탭에서 장소 저장
  → Plan 탭 진입
  → Spot 없음: 빈 상태 (Spots 탭으로 유도)
  → Spot 있음: Spot 가장 많은 도시 자동 선택 → 지도에 클러스터링된 상태 (WOW moment)
  → 다중 도시면 상단 드롭다운으로 도시 전환 가능
  → 일자 조절 → 자동 저장
  → 바텀 시트 드래그로 리스트 뷰 전환 → 편집
  → AI 추천 Spot 수락 or 거절
  → 여행 출발
```

---

## 1. Spots 탭

### 기본 상태
- Reel 썸네일 단위로 2열 그리드 표시 (9:16 비율)
- 각 Reel 안에 포함된 Spot 개수 배지 (예: "3 spots")
- 플랫폼 뱃지: Instagram / TikTok
- **편집 없음** — Spots 탭은 읽기 전용 인박스

### Plan 연결 상태 표시
Reel 썸네일 하단에 Plan 연결 정보를 뱃지로 표시. 유저가 Spots 탭에서 "이 장소가 어느 여행에, 며칠에 들어가 있는지" 바로 확인 가능.

```
┌──────────┐  ┌──────────┐
│          │  │          │
│  [썸네일] │  │  [썸네일] │
│          │  │          │
│ 3 spots  │  │ 2 spots  │
│ 🇺🇸 Day2 │  │ (미연결)  │
└──────────┘  └──────────┘
```

- **Plan에 포함된 Spot이 있는 Reel**: 도시 국기 + Day 정보 뱃지 표시
  - 예: `🇺🇸 Day 2` / `🇰🇷 Day 1`
  - 여러 Day에 걸쳐 있으면: `🇺🇸 Day 1·2`
- **Plan에 없는 Reel**: 뱃지 없음 → 자연스럽게 Plan 탭 진입 유도
- 뱃지 탭 → 해당 Plan의 지도 뷰로 바로 이동

### Reel 클릭
- 해당 Reel에서 추출된 Spot 리스트 확장 표시

### Spot 삭제
- Reel 롱프레스 or 스와이프 → 삭제
- **영구 삭제** — Spots 탭에서 지우면 Plan에서도 사라짐
- 삭제 직후 5초간 Undo 토스트 표시

## 2. Plan 탭

### 빈 상태 (저장된 Spot 없을 때)
- 일러스트 + 안내 문구: "릴스에서 저장한 장소들이 여기서 여행이 돼요"
- CTA: "Spots 탭에서 장소 저장하기" → Spots 탭으로 이동

### Spot이 있을 때 — 진입 즉시 WOW
- Plan 탭에 진입하는 순간, 저장된 Spot들이 **이미 지도에 클러스터링된 상태**로 보임
- 별도 "Plan 만들기" 버튼이나 플로우 없음
- 유저가 탭을 누르는 순간이 곧 WOW moment

---

## 3. Plan 상세

### 도시 자동 선택 (다중 여행지 대응)
- 저장된 Spot들을 geo-clustering으로 자동 도시 감지
- Reverse geocoding으로 도시명 표시
- **Spot이 가장 많은 도시를 자동 선택** → 탭 진입 즉시 지도 WOW
- 단일 도시: 상단 도시명만 표시
- 다중 도시: 상단 드롭다운으로 도시 전환

```
[🇺🇸 San Francisco ∨]   ← 탭하면 다른 도시 선택
  🇰🇷 Seoul
  🇦🇺 Sydney
```

---

### 지도 보기 (기본값 — WOW)

**기본 구성**
- 전체 화면 지도
- 저장된 Spot들이 근접 클러스터링으로 컬러 그룹화
- 일자 자동 배분: 1~4개 → 1일 / 5~8개 → 2일 / 9개+ → 3일
- 상단 세그먼트 컨트롤: **1일 / 2일 / 3일** — 선택 즉시 재계산
- 클러스터 컬러: Day 1 `#FF6B6B` / Day 2 `#4ECDC4` / Day 3 `#A78BFA`
- **자동 저장** — 별도 저장 버튼 없음

**Spot 마커 아이콘 (카테고리별)**
| 카테고리 | 아이콘 |
|---|---|
| 카페 | ☕ |
| 식사 | 🍽️ |
| 자연 / 공원 | ⛰️ |
| 뷰포인트 | 🌉 |
| 해안 / 해변 | 🌊 |
| 쇼핑 | 🛍️ |

**Day AI 요약**
- 지도 하단 Day 탭 칩 아래 강조 한 줄 표시
- Spot 카테고리 조합 기반으로 하루 분위기 서술
- 지명 / 고유명사 사용 금지, 15자 내외, 구어체

```
Day 1  ●  바다를 끼고 걷다가 카페로 마무리하는 날
Day 2  ●  브런치하고 공원에서 쉬어가는 여유로운 하루
```

예시: "바닷바람 맞으며 걷는 하루" / "카페에서 시작해 노을로 끝나는 날" / "시장 구경하고 공원에서 쉬는 오후"

**마커 탭 → 디테일 카드**
- 하단에 카드 슬라이드업: Reel 썸네일 + 이름 + 설명
- **"이 여행에서 빼기" 버튼 포함** — 지도 뷰에서 직접 제외 가능

```
┌─────────────────────────┐
│ 🌉 Golden Gate Bridge   │
│ 브리지 뷰와 베이 전망     │
│                         │
│   [이 여행에서 빼기]      │
└─────────────────────────┘
```

---

### 리스트 보기 (바텀 시트 드래그)

**전환 방식**
- 지도 하단 핸들(—)을 위로 드래그 → 리스트 슬라이드업
- 아래로 드래그 → 지도 전체로 복귀
- 별도 버튼 없음. 레이어 개념으로 지도 위에 리스트가 올라옴

```
지도 전체 화면
━━━━━━━━━━  ← 핸들 (드래그 업)
  Day 1  바다를 끼고 걷다가 카페로 마무리하는 날
  ① Golden Gate  ② Baker Beach  ③ Warming Hut
```

**펼쳐진 상태 레이아웃**
- 상단 40%: 지도 (현재 선택 Spot 하이라이트)
- 하단 60%: 리스트

**리스트 구조**
```
[ Day 1 ]  ●  바다를 끼고 걷다가 카페로 마무리하는 날
  ① Golden Gate Bridge  🌉
  ② Baker Beach         🌊
  ③ Warming Hut Café    ☕

[ Day 2 ]  ●  브런치하고 공원에서 쉬어가는 여유로운 하루
  ① Tartine Bakery      ☕
  ② Dolores Park        ⛰️

────────────────────────
[ AI 추천 ]  (조건 충족 시)
  ┄ Lazy Bear  🍽️  → 추가 / 건너뛰기

────────────────────────
[ 이 Plan에서 제외된 Spot ]  ∨ (접힘)
  · Lands End  ⛰️   →  + 추가
  · Outerlands ☕   →  + 추가
```

**인터랙션**
- 리스트 아이템 탭 ↔ 지도 핀 하이라이트 (양방향)
- **Day 내 순서 변경**: 드래그로 같은 Day 안에서 순서 변경 → 지도 루트 즉시 업데이트
- **Day 간 이동 (primary)**: 드래그로 다른 Day 영역에 드롭 → 해당 Day로 이동
- **Day 간 이동 (fallback)**: Spot 탭 → 디테일 카드 → "Day 변경" 옵션
- 스와이프 왼쪽 → "이 여행에서 빼기" (Spots에는 남음)

---

## 4. 삭제 vs 제외 — 핵심 구분

| 액션 | 위치 | 결과 |
|---|---|---|
| **삭제** | Spots 탭 | 영구 삭제. 모든 Plan에서도 사라짐 |
| **이 여행에서 빼기** | Plan 탭 (지도 디테일 카드 or 리스트 스와이프) | 이 Plan에서만 제외. Spots에 남음 |

제외 시 토스트: *"이 여행에서 제외했어요. Spots에는 남아 있어요."*

## 5. AI 추천 & 갭 감지

### 트리거 조건

**양적 갭** (기존)
- 특정 Day에 Spot이 1개 이하
- 루트 상 이동 구간이 30분 이상 비는 경우

**카테고리 갭** (신규)
- 식사 관련 Spot(🍽️)이 하루에 없을 때 → "점심/저녁 식사 장소가 없어요"
- 카페(☕)만 3곳 이상 → "카페가 많네요. 다른 종류 장소를 추가해볼까요?"
- 실내 카테고리(카페·식사)만 있고 야외(자연·해안·뷰포인트)가 없을 때 → "야외 활동이 없는 하루예요"
- 모든 Spot이 같은 카테고리일 때 → 다양성 제안

### 갭 메시지 위치
- 리스트 뷰 Day 헤더 아래 인라인으로 표시
- Day AI 요약 한 줄 바로 아래

```
[ Day 1 ]  ●  바다를 끼고 걷다가 카페로 마무리하는 날
⚠️ 식사 장소가 없어요 — 근처 식당을 찾아볼까요?  → [추천 보기]
  ① Golden Gate Bridge  🌉
  ② Baker Beach         🌊
  ③ Warming Hut Café    ☕
```

### 추천 카드 표시 방식
- "추천 보기" 탭 → 해당 Day 리스트 하단에 추천 카드 슬라이드다운
- 카드: dashed border, 배경 #F0F0F0, "AI 추천" 뱃지
- 장소명 + 카테고리 + 현재 루트 기준 거리
- "추가하기" → 해당 Day 최적 위치에 삽입 + 지도 반영
- "건너뛰기" → 카드 숨김 (해당 세션 동안, 앱 재진입 시 다시 노출)

### 갭 감지 우선순위
여러 갭이 동시에 감지되면 하나만 노출 (중복 방지)
1. 식사 없음 (가장 중요)
2. Spot 1개 이하
3. 이동 구간 30분 이상
4. 카테고리 단조로움

## 6. 디자인 스펙

**방향**: 블랙 베이스 기능 UI + 컬러는 지도 마커·루트라인에만. 컬러가 버튼처럼 느껴지는 문제 방지.

> **원칙**: 인터랙티브 요소(버튼, 탭, CTA)는 블랙/화이트로. 클러스터 컬러(레드·틸·퍼플)는 지도 마커와 루트라인 전용. Day 칩·배지 등 UI 컴포넌트에 클러스터 컬러 사용 금지.

---

### 컬러

| 용도 | 값 |
|---|---|
| 배경 | `#FFFFFF` |
| 보조 배경 | `#F8F8F8` |
| 텍스트 Primary | `#111111` |
| 텍스트 Secondary | `#666666` |
| 텍스트 Tertiary | `#999999` |
| 구분선 / 테두리 | `rgba(0,0,0,0.08)` |
| 버튼 / 액티브 상태 | `#111111` (블랙) |
| Day 1 클러스터 (지도 전용) | `#FF6B6B` |
| Day 2 클러스터 (지도 전용) | `#4ECDC4` |
| Day 3 클러스터 (지도 전용) | `#A78BFA` |

**글래시 카드 (바텀 시트, 디테일 카드, 네비게이션)**
```css
background: rgba(255, 255, 255, 0.85);
backdrop-filter: blur(20px);
-webkit-backdrop-filter: blur(20px);
border: 1px solid rgba(255, 255, 255, 0.6);
```

---

### 타이포그래피

| 레벨 | 크기 | 굵기 | 용도 |
|---|---|---|---|
| Display | 24px | 700 | 페이지 타이틀 |
| Title | 17px | 600 | 섹션 헤더, 카드 제목 |
| Body | 15px | 400 | 본문, 설명 |
| Caption | 13px | 400 | 보조 정보, 뱃지 |
| Micro | 11px | 500 | 레이블, 태그 |

- 폰트: 시스템 폰트 (`-apple-system, BlinkMacSystemFont, 'Segoe UI'`)
- 줄 간격: 1.4~1.5

| xs | 4px | 아이콘-텍스트 간격 |

| md | 16px | 카드 패딩, 섹션 간격 |
| lg | 24px | 섹션 상하 여백 |
| xl | 32px | 화면 상단 여백 |

**카드**

```css

```


**네비게이션 pill**
```css

/* 글래시 효과 적용 */
```

**Day 칩 (바텀 시트 내)**
```css

padding: 6px 14px;
font-size: 13px;
font-weight: 600;
background: #111; /* 선택 시 블랙, 미선택 시 rgba(0,0,0,0.06) */

/* 클러스터 컬러 사용 금지 */
```

**아이콘 (네비게이션 Plan 탭)**
- 곡선 루트 연결 아이콘 (waypoint 2개를 곡선으로 연결하는 형태)

### 모션

| 용도 | 값 |
|---|---|
| 기본 트랜지션 | `0.25s ease` |
| 바텀 시트 슬라이드 | `0.35s cubic-bezier(0.32, 0.72, 0, 1)` |
| 마커 등장 | `0.3s ease, stagger 80ms` |
| 카드 슬라이드업 | `0.3s cubic-bezier(0.34, 1.56, 0.64, 1)` |

---

### 지도

- 타일: CartoDB Voyager (밝고 깔끔, 클러스터 컬러와 대비 좋음)
- 마커: 원형, 컬러 + 카테고리 이모지
- 루트라인: 클러스터 컬러 solid 2px
- 지도 자체에 UI 크롬 최소화

---

### 레이아웃

- Mobile-first, max-width 430px
- 데스크탑에서도 430px 컨테이너로 센터 정렬
- 바텀 네비 높이: 72px (safe area 포함)
- 콘텐츠 하단 여백: 바텀 네비 높이만큼 확보

## 7. 검증해야 할 핵심 질문

- Plan 탭 진입만으로 WOW가 충분히 전달되는가
- 의미 있는 클러스터가 만들어지는 최소 Spot 임계값은 몇 개인가
- 바텀 시트 드래그가 리스트 진입 방법으로 충분히 발견되는가
- 도시 자동 감지 정확도가 UX에 영향을 주는가

## 7. Edge Cases

| 케이스 | 처리 |
|---|---|
| Spot 0개 | 빈 상태 + "릴스에서 장소를 저장해보세요" + Spots 탭 이동 CTA |
| 위치 정보 없는 Spot | Plan 포함 불가. Spots에는 유지. 리스트 하단 "위치 미확인" 섹션 표시 |
| Plan 전체 Spot 제외 | 빈 상태 + "Spots에서 추가해보세요" 안내 |
| 단일 도시만 존재 | 도시 드롭다운 미노출, 바로 지도 뷰 |
| Spots 탭에서 삭제한 Spot이 Plan에 있던 경우 | Plan에서도 즉시 제거 + 토스트: "Spot이 삭제되어 Plan에서도 제외됐어요" |
| Day 1 Spot을 드래그해 Day 3으로 이동 | 중간 Day 건너뛰기 허용. 드롭한 Day에 추가됨 |

## 8. Out of Scope (이번 버전)

- 예약 / 부킹 연동
- 다른 유저와 Plan 공유
- Explore 탭 → Plan 임포트 연동
- 오프라인 저장
- Plan 당일 알림
- AI 추천 영구 숨기기
- Spot 배틀 (같은 카테고리 중복 시 1:1 비교로 선택) → 다음 버전
